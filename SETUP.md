# Инструкции за стартиране на проекта

## Изисквания

1. **Java 17** - трябва да е инсталиран
2. **MySQL** - трябва да е инсталиран и да работи
3. **Maven** - за компилиране на проектите

## Стъпки за стартиране

### 1. Конфигурация на MySQL

Проектът използва MySQL с:
- **Host**: `localhost`
- **Port**: `3306`
- **Username**: `root`
- **Password**: `123456`

Ако вашата MySQL конфигурация е различна, трябва да промените `application.properties` файловете в двата проекта.

### 2. Стартиране на Rating Service (микросервис)

```bash
cd event-rating-svc
mvn spring-boot:run
```

Сервисът ще стартира на порт **8081** и автоматично ще създаде базата данни `event_rating_db` (ако не съществува).

### 3. Стартиране на основното приложение (EventApp)

В нов терминал:

```bash
cd EventApp
mvn spring-boot:run
```

Приложението ще стартира на порт **8080** и автоматично ще създаде базата данни `event_app_db` (ако не съществува).

### 4. Отваряне на приложението

Отворете браузър и отидете на: `http://localhost:8080`

## Важни бележки

- **Няма нужда от API ключове** - проектът използва само публични API-та:
  - OpenStreetMap Nominatim (за геокодиране) - без нужда от ключ
  - OpenStreetMap tiles (за карти) - без нужда от ключ

- **Базите данни се създават автоматично** благодарение на `createDatabaseIfNotExist=true` в connection string-овете

- **Двата проекта трябва да работят едновременно**:
  - Rating Service на порт 8081
  - EventApp на порт 8080

## Проблеми и решения

### MySQL не работи
- Проверете дали MySQL сървърът е стартиран
- Проверете дали потребителят `root` с парола `123456` съществува
- Ако имате различна конфигурация, променете `application.properties`

### Портът е зает
- Променете `server.port` в `application.properties` файла
- Уверете се, че няма друг процес, който използва портовете 8080 или 8081

### Rating Service не отговаря
- Уверете се, че Rating Service е стартиран преди EventApp
- Проверете дали порт 8081 е свободен
- Проверете логовете за грешки
